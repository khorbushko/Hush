Build a macOS menu bar application called "Hush" — an ambient sound mixer that helps users create a personal acoustic environment to mask distracting or loud surroundings.

---

## PLATFORM & ARCHITECTURE

- macOS 14+ (Sonoma) target
- Language: Swift 5.9+
- UI: AppKit for the menu bar item + NSPopover; SwiftUI for all panel content
- Audio: AVFoundation (AVAudioEngine + AVAudioPlayerNode for mixing)
- Package manager: Swift Package Manager only — no CocoaPods, no Carthage
- Project structure: well-organized with clear separation of concerns (Models, ViewModels, Views, Services, Resources)

---

## ENTRY POINT & MENU BAR BEHAVIOR

- App runs as an agent (LSUIElement = YES in Info.plist — no Dock icon, no main window)
- NSStatusItem with a dynamic SF Symbol icon: "waveform" when playing, "waveform.slash" when stopped
- Clicking the status bar icon opens an NSPopover anchored to the status item
- Popover is transient: clicking outside dismisses it
- App launches at login (offer toggle in Settings)

---

## POPOVER WINDOW

- Size: 320×520pt, no resize
- Built entirely in SwiftUI inside NSHostingView
- Respects system light/dark mode with a manual override toggle
- Visual style: follows macOS Sonoma HIG — use materials (.regularMaterial), native vibrancy, rounded corners (12pt), subtle shadow
- Custom app icon in the header row, title "Hush", version string

## AUDIO ENGINE

- Use AVAudioEngine with one AVAudioPlayerNode per sound
- All nodes connect to a single AVAudioMixerNode → engine output
- Each sound loops indefinitely using scheduleBuffer(_:at:options:completionHandler:) with .loops option
- Preload all audio buffers at launch in a background Task
- Master volume: a top-level slider affecting the engine's mainMixerNode.outputVolume
- Gracefully handle audio interruptions (e.g., phone call via Continuity)

---

## TIMER

- Optional sleep timer: Off / 15 min / 30 min / 45 min / 60 min / Custom
- When active, show a countdown in the header
- When timer expires, fade all sounds out over 3 seconds, then stop the engine
- Custom option opens an inline picker (hours + minutes wheels)

## SETTINGS PANEL

Accessible via a gear icon in the popover header. Implemented as a NavigationStack push inside the popover (not a separate window).

Settings include:
- Launch at login toggle (uses ServiceManagement / SMAppService)
- Default master volume (slider)
- Timer default duration (picker)
- Notification when timer ends (UNUserNotificationCenter)
- "Reset to defaults" button

## ADDITIONAL HEADER CONTROLS

Row of icon buttons at the top of the popover:
- Settings (gear) — navigates to Settings panel
- Theme toggle (sun/moon) — overrides system appearance for the popover only
- About (info.circle) — shows a small sheet: app name, version, credits, a link to a placeholder website
- Quit (power) — calls NSApplication.shared.terminate(nil) after a 0.2s delay

## THEME & APPEARANCE

- Light mode: background uses .regularMaterial, text primary #1C1C1E, accent color a warm sand #C8A97E
- Dark mode: background uses .ultraThinMaterial, accent color a soft amber #E8B86D
- Manual override stored in AppStorage("colorScheme") and applied via .preferredColorScheme()
- Category section headers use SF Pro Rounded (via .fontDesign(.rounded))
- All interactive controls use native SwiftUI Slider and Toggle with custom tint using .tint()

## STATE MANAGEMENT

- Single ObservableObject `SoundMixerViewModel` owns all sound states, master volume, timer state
- Persist active sounds + individual volumes to UserDefaults via AppStorage / @AppStorage where appropriate
- Restore last session on launch (sounds that were playing resume at saved volumes)
